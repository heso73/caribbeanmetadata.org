03

Valider un fichier CMS

Python 
# pip install jsonschema requests
import requests, jsonschema, json

# 1. Charger le schéma CMS officiel
schema_url = "https://raw.githubusercontent.com/heso73/cms-nlp-corpus/main/data/CMS_corpus_schema_v2.json"
schema = requests.get(schema_url).json()

# 2. Charger votre fichier œuvre
with open("mon_oeuvre.json") as f:
    oeuvre = json.load(f)

# 3. Valider
try:
    jsonschema.validate(instance=oeuvre, schema=schema)
    print("✅ Fichier conforme au standard CMS v2.0")
except jsonschema.ValidationError as e:
    print(f"❌ Erreur : {e.message}")

API publique officielle v2.0 LIVE

L'API CMS v2.0 est publique, gratuite et connectée en temps réel à Supabase. Aucune clé requise pour les requêtes en lecture. Données réelles : 110 enregistrements corpus, vocabulaire complet v2.0.

BASE URL — v2.0
https://cms-api.small-disk-3275.workers.dev
v1 → redirigée vers v2 · CORS: * · Format: JSON
MÉTHODE + ENDPOINT DESCRIPTION
GET/api/v2Index — endpoints, version, licence
GET/api/v2/statsStatistiques en temps réel — œuvres, certifications, langues, territoires
GET/api/v2/worksListe des œuvres — filtres: territory, family, language, status, limit, offset
GET/api/v2/works/:idDétail d'une œuvre par UUID — avec créateur
POST/api/v2/worksSoumettre une œuvre pour enregistrement CMS — body JSON requis
GET/api/v2/certificationsRegistre public des certifications CMS
GET/api/v2/certifications/:cms_idVérifier une certification par CMS ID — valid, cms_id, work, completeness
GET/api/v2/vocabulary/markers27 marqueurs culturels officiels CMS — id, name, category, territory
GET/api/v2/vocabulary/languages10 langues caribéennes ISO 639-3
GET/api/v2/vocabulary/territories20 territoires caribéens
GET/api/v2/vocabulary/domains14 domaines culturels

Exemples de requêtes

CURL — Stats en temps réel
curl https://cms-api.small-disk-3275.workers.dev/api/v2/stats
CURL — Œuvres par territoire
curl "https://cms-api.small-disk-3275.workers.dev/api/v2/works?territory=Guadeloupe&limit=10"
JAVASCRIPT — Vérifier une certification
const res = await fetch(
  'https://cms-api.small-disk-3275.workers.dev/api/v2/certifications/CMS-GUA-2026-XXXXXXXX'
);
const cert = await res.json();
console.log(cert.valid, cert.work.title);
PYTHON — Lister les marqueurs
import requests
r = requests.get('https://cms-api.small-disk-3275.workers.dev/api/v2/vocabulary/markers')
markers = r.json()['markers']
for m in markers:
    print(m['id'], '-', m['name'], '(', m['category'], ')')
POST — Soumettre une œuvre

Niveaux de conformité CMS

La complétude d'une fiche CMS se mesure par le nombre de familles de métadonnées renseignées sur 6.

FAMILLES NIVEAU DESCRIPTION
1–2 / 6PartielleFiche démarrée — langue ou territoire identifié
3–4 / 6AvancéeFiche substantielle — langue + territoire + marqueurs documentés
5–6 / 6ComplèteFiche CMS complète — toutes les dimensions caribéennes documentées
6 / 6ExhaustiveFiche de référence — standard maximal CMS v2.0
