The Problem

Caribbean content is invisible — by design

Existing metadata systems were not built for our languages, rhythms, or cultural markers. The result: algorithmic invisibility.

Misclassification

Platforms lack categories for Caribbean genres, dialects, and cultural nuances. Content is forced into foreign frameworks.

Algorithmic Blindness

Recommendation engines cannot promote what they cannot interpret. Invisibility compounds over time.

Cultural Dilution

Without proper metadata, Caribbean identity is flattened into generic or inaccurate categories, erasing specificity.

Technical Interoperability

Built on schema.org, Dublin Core & EBUCore

The CMS is an extension — not a replacement. Three integration levels match any technical capacity.

Level 1

Basic

Accept CMS tags as additional keywords in existing fields (e.g., dc:subject). Requires no infrastructure change.

Hours of work
Level 2

Structured

Map CMS families to dedicated fields or schema:additionalProperty. Enables richer queries and filtering.

Days of work
Level 3

Native

Fully integrate the six CMS families as first-class metadata fields. Unlocks full algorithmic potential and Caribbean discoverability.

Weeks of work 
// CMS-compliant JSON-LD example (v2.0 — corrected)
{
  "@context": {
    "@vocab": "https://schema.org/",
    "cms":    "https://caribbeanmetadata.org/ns/"  // CMS namespace
  },
  "@type":             "CreativeWork",
  "name":             "Anba Dlo",
  "inLanguage":       "ht",              // BCP 47 — required by schema.org
  "keywords":         ["konpa", "haiti", "musique-carib"],
  "cms:territory":    "Haiti",
  "cms:cultural_markers": ["Afro-Caribbean spirituality"],
  "cms:rhythmic_tempo": "syncopated",
  "cms:certification": "Gold"
}

Governance

A regional common good

Not a private standard. Designed to transfer to independent Caribbean governance the moment five institutions adopt it.

Technical Committee

5–9 experts. Validates new tags, corrections, minor versions. 2-year mandates. 1/3 renewed yearly.

General Assembly

All adopting institutions, one vote each. Approves major versions and strategic directions.

Secretariat

Manages daily operations, website, GitHub repository, and certification process.

Transfer clause: As soon as 5 independent institutions officially adopt the CMS — national archives, universities, cultural ministries — governance transfers entirely from Caribwood Language Lab to the independent Technical Committee. The CMS becomes the Caribbean's standard, not any single organization's property.

Roadmap

18 months to a regional standard

Three phases. Each one builds the foundation for the next.

1
Months 0–3 · Foundation

CMS v1.0 + Metadata Bible

Linguistic & cultural committee convened. Six families defined. Glossary published. Manual tagging tools ready.

2
Months 3–9 · Adoption

Pilot platforms & certification launch

First 2–3 institutions onboarded. CMS Tagger prototype live. Bronze/Silver/Gold/Platinum badges issued.

3
Months 9–18 · Scaling

Governance transfer + CMS v2.0 public

AI-assisted tagging (experimental). Independent governance activated. CMS v2.0 published as an open standard.
