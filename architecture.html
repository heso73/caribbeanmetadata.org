Architecture officielle

L'écosystème CMS en 4 couches

Le CMS n'est pas un fichier isolé — c'est un écosystème structuré en couches interdépendantes, de la définition du standard jusqu'à la diffusion mondiale des contenus caribéens.

COUCHE 1 — STANDARD (Le cœur)
Schéma JSON — CMS v1.0 + Schéma XML/XSD + Déclaration Fondatrice + Bible des métadonnées
COUCHE 2 — OUTILS (L'interface)
Validateur CMS + Registre des 100 Premiers + Générateur JSON + API CMS (v2)
COUCHE 3 — DONNÉES (Le contenu)
Métadonnées d'œuvre + Profils créateurs + Registre territoires + Corpus caribéen
COUCHE 4 — DIFFUSION (L'impact)
Plateformes streaming + Archives culturelles + Moteurs de recherche + IA de recommandation
COUCHE 1

Standard

Les fichiers de définition officielle — JSON Schema et XSD — qui constituent la référence normative du CMS. Tout part de là.

COUCHE 2

Outils

Les interfaces permettant aux créateurs et institutions d'interagir avec le standard — valider, certifier, soumettre, interroger.

COUCHE 3

Données

Les enregistrements réels — œuvres taguées, profils créateurs, territoires reconnus — qui constituent le corpus caribéen vivant.

COUCHE 4

Diffusion

Les plateformes, moteurs et systèmes qui consomment les métadonnées CMS pour rendre les contenus caribéens visibles et recommandables.

Mapping avec les standards existants

Correspondance CMS ↔ standards internationaux

Le CMS est conçu pour être interopérable. Chaque champ CMS correspond à un ou plusieurs champs des standards existants — il ne les remplace pas, il les enrichit avec la dimension caribéenne.

Dublin Core (DC)
Standard universel de description de ressources numériques — 15 éléments de base
COMPATIBLE
Champ CMS Équivalent Dublin Core Compatibilité Note
identity.title.original dc:title Totale CMS ajoute les traductions multilingues
identity.creator[].name dc:creator Totale CMS ajoute le rôle, la nationalité et le statut diaspora
content_type dc:type Totale CMS précise avec content_subtype
identity.year_of_release dc:date Totale CMS distingue création et publication
language.primary dc:language Totale CMS ajoute variante créole, langues secondaires, sous-titres
rights.license dc:rights Totale CMS structure les droits par territoire
origin.territory[] dc:coverage Partielle DC couvre lieu + période, CMS spécialise sur les 27 territoires caribéens
cultural_markers.themes[] dc:subject Extension CMS DC est générique — CMS ajoute 18 thèmes caribéens spécifiques
Schema.org
Vocabulaire structuré pour le web sémantique — utilisé par Google, Bing, Yahoo pour l'indexation
COMPATIBLE
Champ CMS Équivalent Schema.org Compatibilité Note
identity.title.original schema:name Totale Applicable à Movie, MusicAlbum, Book selon content_type
identity.creator[] schema:director / schema:author Totale CMS mappe le rôle vers la propriété Schema.org correspondante
content_type + content_subtype schema:@type Totale audiovisual/film → Movie, musical/album → MusicAlbum, literary → Book
language.primary schema:inLanguage Totale Codes BCP-47 partiellement couverts — gcf et pap nécessitent extension
origin.territory[] schema:countryOfOrigin Partielle Schema.org n'a pas de notion de territoire culturel distinct du pays légal
cultural_markers.caribbean_identity_score — aucun équivalent — Exclusif CMS Concept inédit — score d'ancrage culturel caribéen de 0 à 100
cultural_markers.cultural_expressions[] schema:genre Partielle Schema:genre est trop générique — CMS liste 30+ expressions caribéennes nommées
technical.duration_seconds schema:duration Totale Conversion ISO 8601 requise (PT1H37M → 5820s)
ISAN — International Standard Audiovisual Number
Identifiant unique international pour les œuvres audiovisuelles — ISO 15706
COMPLÉMENTAIRE
Champ CMSÉquivalent ISANCompatibilitéNote
identity.isbn_issn_isan ISAN Root Totale Le champ CMS accepte directement le code ISAN comme identifiant externe
identity.title.original ISAN Title Totale Titre de référence identique
identity.year_of_creation ISAN Year Totale Même champ, même format année
cultural_markers + origin — non couvert par ISAN — Extension CMS ISAN identifie mais ne décrit pas culturellement — le CMS comble ce vide
ISRC — International Standard Recording Code
Identifiant unique pour les enregistrements sonores — ISO 3901
COMPLÉMENTAIRE
Champ CMSÉquivalent ISRCCompatibilitéNote
identity.isbn_issn_isan ISRC Code Totale Le champ accepte le code ISRC pour les contenus musicaux (content_type: musical)
identity.creator[role=performer] ISRC Performer Totale CMS structure le rôle performer explicitement
cultural_markers.cultural_expressions[] — non couvert — Extension CMS ISRC n'identifie pas le genre musical caribéen — zouk, gwo ka, compas restent invisibles sans CMS
EBUCore — European Broadcasting Union Core Metadata
Standard de métadonnées pour la radiodiffusion et les archives audiovisuelles professionnelles
COMPATIBLE
Champ CMSÉquivalent EBUCoreCompatibilitéNote
technical.format ebucore:format Totale MP4, MXF, ProRes — formats communs
technical.duration_seconds ebucore:duration Totale Même unité temporelle
technical.aspect_ratio ebucore:aspectRatio Totale Valeurs identiques — 16:9, 4:3, 2.39:1
origin.territory[] + origin.location_shoot[] ebucore:locator / ebucore:publishingPlace Partielle EBUCore couvre la localisation technique, CMS ajoute la dimension culturelle caribéenne
cms_compliance.certified ebucore:rights Extension CMS La certification CMS-Compliant n'a pas d'équivalent EBUCore — concept inédit
language.creole_variant ebucore:language Extension CMS EBUCore ne distingue pas les variantes créoles — EBUCore ne reconnaît pas gcf, pap, srn
